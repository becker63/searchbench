Do a narrow structural pass focused only on two things:
	1.	move the actual localizer-agent code into the agents/ package
	2.	make basic, conservative fixes to the active policy/scoring file and its loader

This is not a broad reorg and not a backend refactor.

Scope

A. Move localizer-agent responsibilities into agents/

Right now the repo has an agents/ package, but part of the real agent logic still lives under localization/. I want the actual LLM/tool-execution layer for localization to live in agents/, not in localization/.

Perform a conservative move such as:
	•	move localization/runtime/agent_runtime.py to agents/localizer.py
	•	move any tiny agent-specific helper types that exist only to support that module into agents/ as well, if that improves cohesion
	•	update imports accordingly

Keep these out of agents/:
	•	localization scoring math
	•	canonical eval-record construction
	•	repo materialization
	•	deterministic batch evaluation logic
	•	business/domain models that are not agent-specific

In other words:
	•	the agent goes in agents/
	•	the localization domain runtime/evaluation/scoring stays in localization/

B. Basic fixes to the active policy/scoring file

Do a small cleanup pass on the active policy file.

Requirements:
	•	preserve the current callable contract/signature
	•	preserve business logic intent
	•	do not redesign the policy system
	•	keep it simple and readable

Specifically:
	1.	Make sure the active policy loader points at the actual current policy file in the repo.
	•	If the loader still references an old filename/path, fix it.
	2.	Improve the active scoring function so it is a bit richer and more informative again.
	•	Use the examples file as guidance for shape/readability
	•	but keep the implementation conservative
	•	do not turn this into a giant rewrite
	3.	Keep the active policy easy for the writer/evaluation flow to load and edit.

Non-goals

Do not:
	•	touch backends/
	•	redesign tracing ownership
	•	redesign state machines
	•	move all evaluation into agents/
	•	change scoring semantics in a sweeping way
	•	add new features
	•	perform a broad architecture rewrite
